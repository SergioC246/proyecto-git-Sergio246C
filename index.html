Biblia de Git y GitHub

Mandamientos intocables

  • Lo que no te toca, no se toca. (Ni lo veas)
  • Nunca se trabaja en Main. (git checkout -b develop)
  • Nunca se trabaja en Develop. (git checkout -b rama a trabajar)
  • Al crear una rama hacer un push a esa rama para crearla en GitHub (git push origin rama)
  • Siempre se hace el push a la rama de trabajo. (git push origin rama)
  • Las ramas de trabajo tienen que tener nombres descriptivos. (navbar, header, main, footer, etc)
  • Los mensajes del commit deben ser descriptivos. (git commit -m "footer done")
  • Antes de un push va un pull de develop. (pull origin develop y luego git push origin rama)
  • Antes de comenzar a trabajar en tu rama hacer un pull de develop. (pull origin develop)
  • Cada vez que alguien modifique develop, hacer un pull. (pull origin develop)

Comandos más usados

  • git remote -v (te dice en que rama estas conectado)
  • git push origin rama (se utiliza para subir la información actulizada)
  • git checkout (se utiliza para cambiar de rama)
  • git checkout -b rama (se utiliza para cambiar y crear una nueva rama)
  • git switch (se utiliza para cambiar a la última rama que estuviste)
  • git branch -a (te dice todas las ramas que hay en el proyecto)
  • git diff (te dice lo que se ha modificado y no se ha agregado)
  • git status (te dice el estado del proyecto)

No nos olvidemos de aplicar siempre las buenas técnicas para que el proyecto quede impecable
